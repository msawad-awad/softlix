ATTENDANCE SYSTEM (BIOMETRIC) — REALISTIC IMPLEMENTATION
================================================================================

IMPORTANT RULE:
- DO NOT store fingerprint data or biometric templates.
- Use biometric auth only to confirm the user identity on device.

Attendance tables:
- attendance_devices:
  id, tenant_id, user_id, device_id, platform, device_public_key, created_at
- attendance_events:
  id, tenant_id, user_id, type [check_in,check_out,break_start,break_end],
  timestamp,
  method [biometric,qr,manual],
  location_lat nullable, location_lng nullable,
  ip, device_id,
  proof_signature (signed payload from device) nullable,
  created_at

Flows:
1) Mobile connector app login -> receives JWT
2) User clicks Check-in
3) App requires biometric unlock (FaceID/TouchID)
4) After success, app sends attendance event with:
   - user_id
   - timestamp
   - geolocation (optional)
   - device_id
   - method="biometric"
   - proof_signature: sign(payload) using device key stored during device registration
5) Server verifies tenant, verifies device, stores event
6) Dashboard shows attendance summary + anomalies (late, missing check-out, location mismatch)

Optional enhancements:
- Geofence (allowed locations in settings)
- QR code check-in (office QR changes daily)
- Anti-fraud:
  - detect timezone mismatch
  - device change alerts
  - repeated check-ins
